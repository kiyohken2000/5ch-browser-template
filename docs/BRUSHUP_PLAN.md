### [T1] CI にテスト・Lint を追加 — ステータス: ✅ 完了 (2026-07-08)。rust-check-linux ジョブに clippy / test ステップを追加。ローカルで両方 green を確認済み (clippy 警告 0、テスト 49 件 pass)

**目的**: 現在の CI (`.github/workflows/ci.yml`) は `cargo check` + フロントビルド + smoke UI のみで、`cargo test` / `cargo clippy` が走っていない。リグレッションを CI で検知できるようにする。

**現状**: ワークスペースに Rust テストが約 50 件あるが CI では実行されていない。clippy はローカルの `/lint` スキル頼み。

**変更内容**:
1. `rust-check-linux` ジョブに以下のステップを追加(Windows ジョブは Vulkan ビルドが重いので追加しない):
   - `cargo test --workspace` (`--ignored` は付けない — ネットワークテストは除外)
   - `cargo clippy --workspace -- -D warnings`
2. clippy が現状のコードで警告を出す場合は、このタスク内で警告を解消する(`#[allow]` の乱用はしない)。

**対象ファイル**: `.github/workflows/ci.yml`、警告解消のため各 crate の `lib.rs`

**完了条件**: CI が green。ローカルで `cargo test --workspace` と `cargo clippy --workspace -- -D warnings` が通ること。

### [T3] src-tauri lib.rs のモジュール分割 — ステータス: ❌ 却下 (2026-07-07)

**目的**: `apps/desktop/src-tauri/src/lib.rs` が 2,602 行・Tauri コマンド 75 個で、「2000 行超まで分割しない」ルールの分割条件を満たしている。機能ドメインごとのモジュールに分割し見通しを改善する。

**変更内容**:
1. 以下のようなドメイン別モジュールに分割する(実際の関数配置を見て調整可):
   - `src/commands/ai.rs` — `ai_*` コマンド群(モデル管理・推論)
   - `src/commands/fetch.rs` — 板・スレ・レス取得系
   - `src/commands/post.rs` — 投稿・スレ立て系
   - `src/commands/store.rs` — お気に入り / NG / 既読 / 設定の永続化系
   - `src/commands/auth.rs` — ログイン系
   - `lib.rs` — `run()`、`generate_handler![]`、共通 state 定義のみ残す
2. **挙動を一切変えない純粋なリファクタリング**とする。コマンド名・シグネチャ・返却型は不変。
3. `generate_handler![]` のコマンド列挙は漏れなく維持(コンパイルエラーで検知できるが、`invoke` 側から呼べなくなる漏れに注意)。
4. `LOGIN_COOKIES` 等の static / Mutex の所在と poison 復旧 (`into_inner()`) の扱いを維持。

**対象ファイル**: `apps/desktop/src-tauri/src/lib.rs` → `src/commands/*.rs` 新設

**完了条件**: `cargo check --workspace` / `cargo test --workspace` が通り、`npx tauri dev` で主要フロー(板一覧 → スレ一覧 → レス表示 → 書き込みダイアログ表示)が動作する。git diff が移動のみであることを確認。完了後 `CLAUDE.md` のリポジトリ構成説明を更新。

**目的**: v0.0.5 で実装 → macOS 起動クラッシュで v0.0.6 撤去した機能の再実装。Windows では動作確認済みだった。

**変更内容**(段階的に安全側へ倒す):
1. まず git 履歴から v0.0.5 の実装と v0.0.6 の撤去 diff を確認し、クラッシュの状況を把握する。
